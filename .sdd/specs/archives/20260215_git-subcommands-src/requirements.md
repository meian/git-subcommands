# 要件定義書

## 機能概要
Git の外部サブコマンドとして利用できるコマンド群を提供し、ブランチ更新・ブランチ選択・マージ済みブランチ整理・直近コミット差分確認を効率化する。
公開する実行スクリプトは `src/` 配下に配置し、`git-***` 形式で提供する。

## ユーザーストーリー
- ユーザーとして、Git の反復的なブランチ操作と差分確認を短いサブコマンドで実行したい。なぜなら日常作業を速く正確に進めたいから。
- メンテナーとして、公開スクリプトを `src/` 配下に集約したい。なぜなら配置規則を統一して保守しやすくしたいから。

## 機能要件
### 要件1：公開スクリプト配置
- ユーザーに公開する Git サブコマンド実行スクリプトを `src/` 配下に配置する。
- 受入基準：
  - [ ] `git-latest` / `git-local-branch` / `git-merged` / `git-last` が `src/` 配下に存在すること
  - [ ] 配置変更後も `git <subcommand>` 形式で実行可能なこと（PATH 設定前提）

### 要件2：git latest [branch]
- 指定したブランチへ切り替え、リモート上の最新状態に更新する機能を提供する。
- git内部のブランチ切り替えは一度のみ行う。
- 受入基準：
  - [ ] `git latest <branch>` 実行時に対象ブランチへ切り替わること
  - [ ] 実行後、対象ブランチがリモートの最新状態へ更新されること
  - [ ] `git switch -` で元のブランチに戻れる状態になっていること

### 要件3：git local-branch [branch-pattern]
- ローカルブランチを対象に、`branch-pattern` で絞り込みしつつ `fzf` で選択し、選択ブランチへ切り替える機能を提供する。
- 受入基準：
  - [ ] `branch-pattern` 指定時に一致するローカルブランチのみが選択候補になること
  - [ ] `branch-pattern` 未指定時は全ローカルブランチが選択候補になること
  - [ ] `fzf` で選択したブランチへ切り替わること

### 要件4：git merged [-clean]
- 現在のブランチにマージ済みのローカルブランチ一覧を表示する機能を提供する。
- `-clean` 指定時は、表示対象のマージ済みローカルブランチを削除する。
- 受入基準：
  - [ ] `git merged` 実行時にマージ済みローカルブランチ一覧が出力されること
  - [ ] `git merged -clean` 実行時に対象ブランチが削除されること

### 要件5：git last [-n]
- 最後のコミットの変更差分を表示する機能を提供する。
- `-n`（正の整数）指定時は、n 個前のコミットの変更差分を表示する。
- 受入基準：
  - [ ] `git last` で最新コミットの差分が表示されること
  - [ ] `git last -1`, `git last -2` のように指定したとき、対応する過去コミットの差分が表示されること

## 非機能要件
- パフォーマンス：日常的なローカルリポジトリ操作で実用的な応答時間（対話操作を阻害しない速度）を維持すること。
- セキュリティ：機密情報（API キー、パスワード等）をコード・ログ・要件文書に含めないこと。
- 依存関係：`git local-branch` は `fzf` を必須とし、未導入環境では実行不可であることを明確に通知すること。
