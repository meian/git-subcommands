# 要件定義書

## 機能概要
`git-subcommands` をユーザー環境へ安全かつ再現可能に導入・更新・削除できるように、ルートに `install.sh` と `uninstall.sh` を提供する。`install.sh` は GitHub `main` から HTTPS 経由で取得して `sh` にパイプ実行でき、`~/.local/share/git-subcommands` 配下への配置と、bash/zsh の対話モード設定（`~/.git-subcommands.rc` 読み込み、`~/.local/share/git-subcommands/src` の PATH 追加）を行う。

## ユーザーストーリー
- ユーザーとして、`git-subcommands` をワンライナーでインストール・更新・アンインストールしたい。なぜなら手動作業を減らし、環境構築を再現可能にしたいから。

## 機能要件
### 要件1：インストーラーの追加と基本インストール
- ルートディレクトリに `install.sh` を追加し、GitHub の `main` ブランチから HTTPS ダウンロードして `sh` パイプで実行できること。
- インストール先は `~/.local/share/git-subcommands` とし、初回インストール時に必要なディレクトリを作成すること。
- 受入基準：
  - [ ] `curl/wget` 経由で取得した `install.sh` を `sh` パイプ実行してインストールが完了すること
  - [ ] 初回実行で `~/.local/share/git-subcommands` が作成されること

### 要件2：再実行時の挙動と更新機能
- 既にインストール済みの場合、`--update` が未指定なら処理を行わず終了すること。
- `--update` 指定時は対象ブランチの最新を取得して更新すること。
- 受入基準：
  - [ ] インストール済み環境で `install.sh` 再実行時、`--update` 未指定なら no-op で終了すること
  - [ ] `install.sh --update` 実行時に対象ブランチの最新状態へ更新されること

### 要件3：インストール対象の指定
- `--branch <branch>` でインストール対象ブランチを指定できること。
- `--tag <tag>` でインストール対象タグを指定でき、未指定時はデフォルトブランチを対象にすること。
- インストール済みであっても、指定タグが現在と異なる場合は再インストールすること。
- 受入基準：
  - [ ] `--branch` 指定時に指定ブランチの内容でインストールされること
  - [ ] `--tag` 指定時に指定タグの内容でインストールされること
  - [ ] 既存インストールとタグが異なる場合、`--update` 未指定でも再インストールされること

### 要件4：シェル設定の反映
- `~/.git-subcommands.rc` を作成し、`~/.local/share/git-subcommands/src` を PATH に含める処理を定義すること。
- bash と zsh の対話モード設定ファイルに、`~/.git-subcommands.rc` を読み込む設定を追加すること。
- 正常終了後に `source` 再実行またはシェル再起動を促すメッセージを出力すること。
- 受入基準：
  - [ ] `~/.git-subcommands.rc` が作成されること
  - [ ] bash/zsh の対話モード設定に `~/.git-subcommands.rc` 読み込み設定が追加されること
  - [ ] 完了時に再読み込みまたは再起動案内が表示されること

### 要件5：アンインストーラーの追加
- ルートディレクトリに `uninstall.sh` を追加し、`~/.local/share/git-subcommands` と `~/.git-subcommands.rc` を削除できること。
- インストール時に変更した bash/zsh の対話モード設定を元に戻せること（どちらか一方のみ設定済みでもクリーンアップ可能）。
- 受入基準：
  - [ ] `uninstall.sh` 実行で `~/.local/share/git-subcommands` が削除されること
  - [ ] `~/.git-subcommands.rc` が削除されること
  - [ ] bash/zsh の設定変更箇所が適切にクリーンアップされること

### 要件6：README への手順追記
- `README.md` にインストール・アンインストール手順を追記すること。
- bash/zsh の対話モード設定ファイルが変更されること、および動作環境として bash/zsh をサポートすることを明記すること。
- 受入基準：
  - [ ] README にインストール手順が記載されること
  - [ ] README にアンインストール手順が記載されること
  - [ ] README に bash/zsh 対応と対話モード設定変更の説明があること

## 非機能要件
- パフォーマンス：インストール・更新・アンインストールは不要な再実行を避け、通常の開発環境で短時間に完了すること。
- セキュリティ：シェル設定の変更は対象箇所を限定し、既存ユーザー設定を不必要に破壊しないこと。
