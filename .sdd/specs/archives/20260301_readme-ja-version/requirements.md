# 要件定義書

## 機能概要
本機能は、`README.md` と同等の情報を日本語で提供する `README.ja.md` を整備し、英語版・日本語版の相互参照と同期運用ルールを定義することを目的とする。
これにより、利用者は希望言語で同一内容を参照でき、メンテナーはドキュメント差分の発生を防止できる。

## ユーザーストーリー
- 日本語利用者として、プロジェクトの説明と利用手順を日本語で読みたい。なぜなら英語版と同等の情報を母語で理解したいから。
- メンテナーとして、英語版と日本語版の README を同期して保守したい。なぜなら言語間の情報差分による誤解や運用ミスを防ぎたいから。

## 機能要件
### 要件1：日本語READMEの整備
- `README.md` と同等内容を日本語化した `README.ja.md` をリポジトリルートに配置する。
- コマンド名、ファイルパス、識別子は原文のまま維持する。
- 受入基準：
  - [ ] `README.ja.md` が存在すること
  - [ ] `README.md` の主要セクション（概要、Subcommands、Requirements、Setup、Test）に対応する日本語セクションがあること

### 要件2：README相互リンク
- 英語版 README と日本語版 README の先頭付近に、相互に遷移できるリンクを配置する。
- 受入基準：
  - [ ] `README.md` から `README.ja.md` へのリンクが存在すること
  - [ ] `README.ja.md` から `README.md` へのリンクが存在すること

### 要件3：同期運用ルールの明文化
- プロジェクトレベルの Codex 向けプロンプトファイル（`AGENTS.md`）に、片方の README 更新時はもう片方も更新するルールを記載する。
- 受入基準：
  - [ ] `AGENTS.md` に README 同期更新ルールが記載されていること
  - [ ] ルールが `README.md` と `README.ja.md` の双方を対象として明示されていること

## 非機能要件
- パフォーマンス：ドキュメント更新のみであり、既存 CLI 実行性能に影響を与えないこと。
- セキュリティ：機密情報（トークン、個人情報、内部URL等）を README および `AGENTS.md` に記載しないこと。
