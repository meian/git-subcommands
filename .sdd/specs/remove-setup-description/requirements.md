# 要件定義書

## 機能概要
インストーラー追加および `src/git-*` 実行権限ルールの導入により不要になったセットアップ記述を、`README.md` と `README.ja.md` から削除し、利用者が現行の導入フローを誤解しないドキュメント状態にする。

## ユーザーストーリー
- ユーザーとして、現行運用に不要なセットアップ手順をREADMEから除外したい。なぜなら古い手順が残っていると導入時に誤った作業をしてしまうから。

## 機能要件
### 要件1：不要セットアップ記述の削除
- `README.md` と `README.ja.md` から、インストーラー運用と実行権限ルールの前提により不要となったセットアップ説明を削除する。
- 受入基準：
  - [ ] 両READMEに不要なセットアップ記述が残っていないこと
  - [ ] 削除後の導入手順が現在のインストール/アンインストール運用と矛盾しないこと

### 要件2：README間の内容整合
- README運用ルールに従い、`README.md` と `README.ja.md` の見出し構成・コマンド例・オプション説明・注意事項を同等に維持する。
- 受入基準：
  - [ ] 英語版/日本語版で該当セクションの情報差分が言語表現差に限定されること
  - [ ] 片方だけに残る削除漏れや情報追加がないこと

### 要件3：影響範囲の限定
- 本対応はドキュメント更新に限定し、`src/`、`lib/`、`test/` の実装・挙動を変更しない。
- 受入基準：
  - [ ] 変更ファイルがREADMEとSDD関連ファイルに限定されること
  - [ ] 既存サブコマンド機能（latest/local-branch/merged/last）に仕様変更が発生しないこと

## 非機能要件
- パフォーマンス：CLI実行性能への影響はないこと（ドキュメント変更のみ）。
- セキュリティ：認証情報・秘密情報をREADMEおよびSDD文書へ追加しないこと。
